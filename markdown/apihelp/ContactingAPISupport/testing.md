# testing

### Testing procedure

Provide any details on how to run your code:

- Names of the documents that must be open
- Names of the features or entities that must be
  selected before running your code, if any
- Whether any bodies or features are suppressed
  or hidden
- Whether an assembly document is loaded
  as lightweight or fully resolved
- Any necessary arguments, parameters, or modifications
  that are necessary to run the code; for example, file pathnames
- Desired output of the code as well as your observed
  output
