# Arrays

This topic contains some
general information about using arrays in SOLIDWORKS.

- In the Microsoft .NET Framework, arrays derive from Object. See the

  [Print Drawing as High Quality (C#)](sldworksapi.chm::/Print_Drawing_as_High_Quality_Example_CSharp.htm)

  and

  [Pack
  and Go an Assembly (C#)](sldworksapi.chm::/Pack_and_Go_an_Assembly_Example_CSharp.htm)

  for examples.
- Passing an
  array of elements of the wrong size to SOLIDWORKS might not do anything
  and might generate an error. For details about longs vs. integers, see

  [Long vs. Integer](Long_vs_Integer.htm)

  .
- See

  [Unmanaged C++
  and C++/CLI Code Differences](Unmanaged_CPP_and_CPP_CLI_Code_Differences.htm)

  for details on handling arrays in unmanaged
  and managed C++.
- See

  [IDispatch Object
  Arrays as Input in .NET](IDispatch_Object_Arrays_as_Input_in_.NET.htm)

  for details on using the DispatchWrapper object
  for non-standard objects like SOLIDWORKS interfaces, which are manipulated
  through the IDispatch interface.
- See the
  following topics for details about double arrays and integer pairs:

  - [Unpacking in VB.NET and VBA](Unpacking_Double_Arrays_into_Integer_Pairs_in_Visual_Basic_.NET_and_Visual_Basic.htm)
  - [Unpacking in C++](PackedIntegersCPlusPlus.htm)
  - [Unpacking in C#](Unpacking_Double_Arrays_into_Integer_Pairs_in_C_.htm)
- See the
  following topics for details about SafeArrays:

  - [Passing SafeArrays in C++](SafeArrays.htm)
  - [Passing SafeArrays in VB](SafeArraysVB.htm)
