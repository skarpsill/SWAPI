# Obsolete SolidWorks APIs

SolidWorks APIs are continually enhanced. In most instances, when an
existing API is superseded by a new API, the new API has the same name
as the existing API followed by a number. A higher number indicates a
more recent API. For example, the methodEntity::Select4supersededEntity::Select3,
which supersededEntity::Select2.
In this example,Entity::Select3,Entity::Select2andEntity::Selectare obsolete APIs.

Obsolete APIs are not supported. You cannot retrieve Help topics for
obsolete APIs using theIndextab. You can, however, retrieve them using either theContentsorSearchtab.

It is recommended that you use the newest APIs in your code, and, whenever
possible, update your existing code to use the newest APIs.
