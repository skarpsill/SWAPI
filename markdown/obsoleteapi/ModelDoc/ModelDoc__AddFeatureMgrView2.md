# ModelDoc::AddFeatureMgrView2

This method is obsolete
and has been superseded by[ModelDoc::AddFeatureMgrView3](ModelDoc__AddFeatureMgrView3.htm).

Description

If you have an existing CView, this method adds that view as a new FeatureManager
design tree view. This method also provides a parameter for adding a tab,
which the user can click to activate your view.

Syntax (OLE Automation)

retval = ModelDoc.AddFeatureMgrView2
( bitmap, appView, toolTip)

(Table)=========================================================

| Input: | (long) bitmap | Pointer to the bitmap you wish to use for the FeatureManager design
tree tab; the bitmap should be no larger than 16x18; this standard pointer
is created by performing a New on CBitmap |
| --- | --- | --- |
| Input: | (long) appView | Pointer to an existing view that you wish to use as your FeatureManager
design tree view; if you do not have an existing view, then see ModelDoc::CreateFeatureMgrView,
which will return a view handle for you |
| Input: | (BSTR) toolTip | ToolTip string |
| Return: | (BOOL) retval | TRUE if the FeatureManager design tree view tab was added successfully,
FALSE otherwise |

Syntax (COM)

status = ModelDoc->AddFeatureMgrView2
( bitmap, appView, toolTip, &retval )

(Table)=========================================================

| Input: | (long) bitmap | Pointer to the bitmap you wish to use for the FeatureManager design
tree tab; the bitmap should be no larger than 16x18; this standard pointer
is created by performing a New on CBitmap |
| --- | --- | --- |
| Input: | (long) appView | Pointer to an existing view that you wish to use as your FeatureManager
design tree view; if you do not have an existing view, then see ModelDoc::CreateFeatureMgrView,
which will return a view handle for you |
| Input: | (BSTR) toolTip | ToolTip string |
| Output: | (VARIANT_BOOL) retval | TRUE if the FeatureManager design tree view tab was added successfully,
FALSE otherwise |
| Return: | (HRESULT) status | S_OK if successful |

Remarks

The FeatureManager design tree view added to this document is not persistent,
that is, it is not stored with this document and must be recreated upon
reloading the document.

This method will not be set up to receive FeatMgrView ActivateNotify
and DeactivateNotify events.

This view was generated by you and can be drawn into as you so desire.
However, because you generated this view, you are responsible for deleting
it. See ModelDoc2::DeleteFeatureMgrView for deleting your Feature Manager
design tree view.
