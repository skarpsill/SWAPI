# ModelDoc::GetRayIntersectionsPoints

This
method is obsolete and has been superseded by[ModelDoc2::GetRayIntersectionsPoints](sldworksAPI.chm::/ModelDoc2/ModelDoc2__GetRayIntersectionsPoints.htm).

Description

This method returns the intersection-point
information generated by ModelDoc::RayIntersections.

Syntax (OLE Automation)

retval = ModelDoc.GetRayIntersectionsPoints ( )

(Table)=========================================================

| Return: | (VARIANT) retval | SafeArray containing the hit points |
| --- | --- | --- |

Syntax (COM)

status = ModelDoc->IGetRayIntersectionsPoints
( retval )

(Table)=========================================================

| Output: | (double*) retval | Pointer to an array containing the hit points |
| --- | --- | --- |
| Return: | (HRESULT)status | S_OK if successful |

Remarks

The array returned contains information about each
hit point of each ray on the given bodies, in the following format:

For each given ray:

(long) nIntersections: number
of intersections found for the ray

For each intersection:

(long) bodyIndex: index in bodiesIn
array of the body that was hit

(long) intersectionType: type of intersection
found. See swRayPtsResults_e. Entry and entry information (swRayPtsResultsENTER
and swRayPtsResultsEXIT) are onlykadov_tag{{<spaces>}}kadov_tag{{</spaces>}}returned
when swRayPtsOptsENTRY_EXIT is specified in the options argument in ModelDoc::RayIntersections).
If more than one intersection type is found, then the intersectionType
will contain an OR'd set of values from the swRayPtsResults_e enumeration.

(double) x,y,z: coordinates of hit
point

(double) nx,ny,nz:normal (Optionally appears
when swRayPtsOptsNORMALS is specified in the options argument in ModelDoc::RayIntersections.)

If normal information is not generated, then the
array returned in this function is 3 elements smaller per intersection
(that is, the array is compacted).
