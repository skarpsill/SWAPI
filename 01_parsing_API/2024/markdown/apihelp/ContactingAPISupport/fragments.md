# fragments

### Avoid code fragments

Code cannot be effectively analyzed or discussed without a small calling/host/wrapper/demonstration
function.

To successfully analyze a problem, API Support must be able to see how
your code is called, where it gets its input data, and how it gets that
data. If possible, please create and send an example that can run as-is
instead of a snippet of code that has been copied and pasted from a complex
application.
