# IGetModuleNames Method (IMacroFeatureData)

Gets the names of the modules in the macro for the macro feature.

## Syntax

### Visual Basic (Declaration)

```vb
Sub IGetModuleNames( _
   ByVal ModuleCount As System.Integer, _
   ByRef ModuleNames As System.String _
)
```

### Visual Basic (Usage)

```vb
Dim instance As IMacroFeatureData
Dim ModuleCount As System.Integer
Dim ModuleNames As System.String

instance.IGetModuleNames(ModuleCount, ModuleNames)
```

### C#

```csharp
void IGetModuleNames(
   System.int ModuleCount,
   out System.string ModuleNames
)
```

### C++/CLI

```cpp
void IGetModuleNames(
   System.int ModuleCount,
   [Out] System.String^ ModuleNames
)
```

NOTE:

See

[Differences Between Unmanaged C++ and C++/CLI Code](DifferencesBetweenUnManagedAndCPPCLI.htm)

.

### Parameters

- `ModuleCount`: Number of modules in the macro
- `ModuleNames`: - in-process, unmanaged C++: Pointer to an array of the names of the modules of size ModuleCount

VBA, VB.NET, C#, and C++/CLI: Not supported

See[In-process Methods](sldworksapiprogguide.chm::/OVERVIEW/In-process_Methods.htm)for details about this type of method.

## Remarks

Before calling this method, call

[IMacroFeatureData::GetModuleCount](SOLIDWORKS.Interop.sldworks~SOLIDWORKS.Interop.sldworks.IMacroFeatureData~GetModuleCount.html)

to determine the size of the array.
