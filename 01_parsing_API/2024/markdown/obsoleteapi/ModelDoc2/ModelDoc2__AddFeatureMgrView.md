# ModelDoc2::AddFeatureMgrView

This
method is obsolete and has been superseded by[ModelDoc2::AddFeatureMgrView2](ModelDoc2__AddFeatureMgrView2.htm).

Description

If you have an existing CView, this method adds that view as a new FeatureManager
design tree view. This method also provides a parameter for adding a tab,which
the user clicks to activate your view.

Syntax (OLE Automation)

retval = ModelDoc2.AddFeatureMgrView
( bitmap, appView)

(Table)=========================================================

| Input: | (long) bitmap | Pointer to the bitmap for the FeatureManager design tree tab; the bitmap
should be no larger than 16x18; this standard pointer is created by performing
a New on CBitmap |
| --- | --- | --- |
| Input: | (long) appView | Pointer to an existing view to use as your FeatureManager design tree
view; if you do not have an existing view, then call ModelDoc2::CreateFeatureMgrView,
which returns a view handle for you |
| Return: | (BOOL) retval | TRUE if the FeatureManager design tree view tab is added successfully,
FALSE otherwise |

Syntax (COM)

status = ModelDoc2->AddFeatureMgrView
( bitmap, appView, &retval )

(Table)=========================================================

| Input: | (long) bitmap | Pointer to the bitmap for the FeatureManager design tree tab; the bitmap
should be no larger than 16x18; this standard pointer is created by performing
a New on CBitmap |
| --- | --- | --- |
| Input: | (long) appView | Pointer to an existing view to use as your FeatureManager design tree
view; if you do not have an existing view, then call ModelDoc2::CreateFeatureMgrView,
which returns a view handle for you |
| Output: | (VARIANT_BOOL) retval | TRUE if the FeatureManager design tree view tab was added successfully,
FALSE otherwise |
| Return: | (HRESULT) status | S_OK if successful |

Remarks

If you have an existing CView, then this method adds that view as a
new FeatureManager design tree view. This method also provides a parameter
for adding a tab, which the user clicks to activate your view.

The FeatureManager design tree view added to this document is not persistent.
In other words, the FeatureManager design tree view is not stored with
this document and must be recreated upon reloading the document.

This method is not set up to receive FeatMgrView::ActivateNotify and
DeactivateNotify events.

This view was generated by you and can be drawn into as you so desire.
However, because you generated this view, you are responsible for deleting
it. See ModelDoc2::DeleteFeatureMgrView for details on how tokadov_tag{{<spaces>}}kadov_tag{{</spaces>}}your
FeatureManager design tree view.
