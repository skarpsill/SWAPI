# ModelDoc::GetRayIntersectionsTopology

This
method is obsolete and has been superseded by[ModelDoc2::GetRayIntersectionsTopology](sldworksAPI.chm::/ModelDoc2/ModelDoc2__GetRayIntersectionsTopology.htm).

Description

This method returns the topology intersections
generated by ModelDoc::RayIntersections.

Syntax (OLE Automation)

retval = ModelDoc.GetRayIntersectionsTopology (VARIANT* retval)

(Table)=========================================================

| Return: | (VARIANT) retval | SafeArray of Dispatch pointers containing the hit entities |
| --- | --- | --- |

Syntax (COM)

status = ModelDoc->IGetRayIntersectionsTopology
( retval )

(Table)=========================================================

| Output: | (LPUNKNOWN*) retval | Array of IUnknown pointers containing the hit
entities |
| --- | --- | --- |
| Return: | (HRESULT) status | S_OK if successful |

Remarks

This method only returns information if the swRayPtsOptsTOPOLS
option was specified in the options argument of ModelDoc::RayIntersections.

The array returned contains information about each
entity hit of each ray on the given bodies. The topology hit can be coordinated
with the hit point using the bodyIndex return value from ModelDoc::GetRayIntersectionsPoints.

For the OLE interface, each item will be an LPDISPATCH.
The type of item hit (face, edge, or vertex) can be determined by checking
the intersectionType return value in the ModelDoc::GetRayIntersectionsPoints.
With this information, you will know which object class (Face, Edge, or
Vertex) to use with the Dispatch pointer.

For the COM interface, an array of LPUNKNOWNs is
returned. The type of item hit (face, edge, or vertex) can be determined
by checking the intersectionType return value in ModelDoc::GetRayIntersectionsPoints,
or by using the LPUNKNOWN pointer in a call to QueryInterface to see which
interface is supported.
