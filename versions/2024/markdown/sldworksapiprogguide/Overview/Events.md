# Events

To receive events, your application must register for the events by
object type. This registration must be done for each instance of a particular
object.

For example, to receive IPartDoc events, your application needs to register
with each IPartDoc object in existence (currently open part files). It
also needs to register with any future IPartDoc objects that are created
(subsequently opened or created part files). See the code generated by
the[SOLIDWORKS
add-in wizards](SolidWorks_API_Add-Ins%2c_Project_Templates%2c_and_Wizards.htm).

The following SOLIDWORKS objects support events:

- [IAssemblyDoc](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IAssemblyDoc.html)
- [IDrawingDoc](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IDrawingDoc.html)
- [IFeatMgrView](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IFeatMgrView.html)
- [IModelView](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IModelView.html)
- [IMotionStudy](swmotionstudyapi.chm::/SolidWorks.Interop.swmotionstudy~SolidWorks.Interop.swmotionstudy.IMotionStudy.html)(not supported by SOLIDWORKS Visual Studio 2005 add-in wizards)
- [IMouse](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IMouse.html)(not supported by SOLIDWORKS Visual Studio 2005 add-in wizards)
- [IPartDoc](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IPartDoc.html)
- [ISldWorks](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.ISldWorks.html)
- [ISWPropertySheet](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.ISWPropertySheet.html)
- [ITaskpaneView](sldworksAPI.chm::/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.ITaskpaneView.html)(not supported by SOLIDWORKS Visual Studio 2005 add-in wizards)

Code for all of these events, unless otherwise
noted, can be automatically generated by the SOLIDWORKS add-in wizards.
For an overview of how to program events, see[Programming Events](Programming_Events.htm).

NOTE:Events are implemented with delegates in the Microsoft .NET Framework.
