# testfiles

### Test files

When possible, send the smallest and simplest drawings, assemblies,
or parts that demonstrate your problem. Most problems can be reproduced
using a small example.

Include all external references used by your drawings, assemblies, and
parts. If missing document references make it impossible to diagnose your
problem, then you will be asked to send a new set of test documents.

If your problem is a regression from a previous release of the software,
then provide either the old version of the file or the old and new versions
of the files that your code references. We must have your files to diagnose
the problem.
